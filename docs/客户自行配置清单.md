## 三、客户拿到源码后需要做的（按顺序）

### 第 1 步：申请第三方账号（1~3 周）
- [ ] 阿里云企业实名认证 → 开通短信服务 + OSS
- [ ] 微信商户平台 → 申请微信支付商户号
- [ ] 支付宝企业版 → 申请企业支付账号
- [ ] Apple Developer（公司版）→ 等待邓白氏编码（约 5~7 天）

> ⚠️ **Apple IAP 上架注意事项**：
> - 虚拟内容（笔记）必须走 Apple IAP，苹果抽 30%
> - 需要在 App Store Connect 中创建商品（product_id）
> - 需要在 Xcode 中配置 StoreKit

### 第 2 步：填写配置文件（30 分钟）

复制 `backend/.env.example` 为 `backend/.env`，填写以下字段：

```env
# 阿里云短信（必填）
SMS_MOCK=false
SMS_ACCESS_KEY_ID=你的阿里云 AccessKey
SMS_ACCESS_KEY_SECRET=你的阿里云 Secret
SMS_SIGN_NAME=你的短信签名（如"心事盒"）
SMS_TEMPLATE_CODE=你的短信模板CODE

# 微信支付（安卓端必填）
WECHAT_APPID=你的微信 APPID
WECHAT_MCHID=你的微信商户号
WECHAT_KEY=你的微信 API 密钥

# 支付宝（安卓端必填）
ALIPAY_APPID=你的支付宝 APPID
ALIPAY_PRIVATE_KEY=你的支付宝私钥
ALIPAY_PUBLIC_KEY=你的支付宝公钥

# Apple IAP（iOS 端必填）
APPLE_SHARED_SECRET=你的 Apple Shared Secret

# 阿里云 OSS（图片存储，可选）
OSS_REGION=oss-cn-hangzhou
OSS_ACCESS_KEY_ID=你的阿里云 Key
OSS_ACCESS_KEY_SECRET=你的阿里云 Secret
OSS_BUCKET=你的 Bucket 名称
```

### 第 3 步：重新编译安装包

```bash
# 安卓
cd mobile && flutter build apk --release

# iOS（需要 Mac + Xcode）
cd mobile && flutter build ios --release
```

### 第 4 步：部署上线

```bash
# 使用 Docker 一键部署到云服务器
docker-compose up -d
```

## 五、常见问题

**Q：客户不会填配置怎么办？**
A：每个 `.env` 字段都有注释说明获取方式，客户按注释去对应平台复制粘贴即可。实在不会可以远程协助。

**Q：客户不想申请这么多账号，能不能简化？**
A：可以。最少只需要：
- 阿里云短信（用于登录）
- Apple Developer + IAP（iOS 上架必需）
- 微信支付或支付宝二选一（安卓支付）

**Q：开发阶段 mock 的数据能保留到生产环境吗？**
A：不行。mock 模式的支付/短信都是假数据，上线前必须关闭 mock，切到真实接口。
